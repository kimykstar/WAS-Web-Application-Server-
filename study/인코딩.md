# 인코딩이란 무엇인가?

정보를 `특정 형식으로 변환하는 과정`이다.

정보를 특정 형식으로 변환하는 목적은 다음과 같다.

1. **데이터 저장**: 정보를 컴퓨터가 이해할 수 있는 형태로 변환한다.
2. **데이터 전송**: 정보를 안전하게 전송하기 위해 특정 형식으로 변환한다.
3. **문자 인코딩**: 텍스트를 컴퓨터가 처리할 수 있는 숫자 형태로 변환한다.(ex. ASCII, UTF-8)
4. **압축**: 데이터의 크기를 줄이기 위해 정보를 다른 형식으로 변환한다.
5. **암호화**: 정보를 보호하기 위해 읽을 수 없는 형태로 변환한다.

## 인코딩의 종류

1. ASCII인코딩(7bit)

- 미국 정보교환 표준 코드로 영문자를 컴퓨터가 해석할 수 있는 숫자로 변환시켜준다.
- 1Byte(8bit) 중 7bit를 사용하여 0 ~ 127까지 총 128개의 문자를 구성한 코드이다.
- 표준은 7bit이나, `확장 ASCII`에서는 256개의 문자를 표현하며 8bit로도 표현이 된다.
- 현대 많은 시스템에서는 8bit로 저장되는 경우도 많다.

2. URL인코딩

- ASCII코드에 없는 영어를 제외한 외국어(한국어 포함)와 ASCII코드에서 표현하지 않는 특수문자를 표현하기 위해 사용된다.
- 한글의 경우 `UTF-8`을 사용한다.

3. HTML인코딩

- HTML문서 안에는 스크립트와 같이 특수한 기능을 하는 문자들이 포함된다. 악의적인 특수기능을 막기 위해 인코딩하여 안전하게 HTML문서와 함께 사용하기 위해서 HTML인코딩을 사용한다.
- XSS의 방어 대책으로 사용된다.

4. BASE64 인코딩

- 2진 데이터를 ASCII형태의 텍스트로 표현할 수 있다.
- 이메일 사용을 위한 인터넷 표준 포멧인 MIME를 사용하기 위해 사용한다.
- BASE64인코딩은 6bit단위로 바이너리를 끊어서 처리하며, 6bit로 끊어진 바이너리 데이터는 ASCII코드로 변환함과 동시에 2bit의 패딩을 붙여 8비트가 된다.
- 6bit단위로 데이터를 끊는 이유는 2^6 = 64 즉, 64개의 안전한 문자로 바이너리 데이터를 표현하기 위함에 있다.

5. ANSI(8bit)

- 8bit로 구성되어있으며, 256개의 문자를 표현할 수 있다.
- ASCII의 확장판으로, ASCII에서 1bit를 더 사용한다. 따라서, ASCII의 7bit를 앞에서 사용하고, 나머지 1bit를 이용해 다른 언어의 문자를 표현한다.
- 각 언어별로 Code값을 주고, Code마다 다른 문자열 표를 의미하도록 약속을 했다.

`ANSI = ASCII(7bit) + CodePage(1bit)`

6. UTF-8

- 유니코드를 위한 가변길이 문자 인코딩(멀티바이트) 방식 중 하나로, ANSI의 단점을 보완하기 위해 만들어졌다.
- 멀티바이트란 표현해야 하는 문자에 따라 글자 크기를 가변적으로 변경하여 사용하는 것을 말한다.
- ANSI는 (1byte = 8bit)로 256자까지만 표현이 가능하나, UTF-8은 멀티바이트(1 ~ 4bytes)로 최대 1,12,064자 까지 표현이 가능하다.
- **첫 128자는 ASCII코드 값으로 ANSI와 UTF-8이 동일하다. 따라서, 영어를 사용할 경우 1byte인 것이다.**
