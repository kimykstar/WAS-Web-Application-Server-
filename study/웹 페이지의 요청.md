# 웹 페이지 요청

웹 페이지를 client에서 요청하면 해당 웹 페이지는 `html`로 작성되었을것이다.

웹 페이지에서는 해당 웹 페이지의 구성을 나타내고 있으며, 해당 코드에는 `link`, `script`등 정적파일들을 참조하는 태그들이 존재한다.

이런 `정적파일`들은 웹 페이지를 요청하는 방식과 마찬가지로 server에게 요청을 해서 client에서 받아야 이미지와 style, js파일 등을 적용할 수 있게 된다.

## 웹 페이지의 요청 과정

`/signup.html`경로를 예로 들어보자.

1. 클라이언트는 url로 `/signup.html`페이지를 요청한다.
2. client의 웹 브라우저는 server로부터 `signup.html`파일을 내려받는다.
3. 웹 브라우저는 HTML파일을 분석하며 CSS와 JS파일에 대해 추가 요청을 보낸다.
4. 서버는 추가 요청에 대한 정적 파일들을 응답한다.

결론: `사용자`는 url을 통해 요청을 1번 보내지만, `웹 브라우저`는 사용자의 요청 그리고 HTML내의 정적파일들을 요청하고 내려받아 페이지를 렌더링하게 된다.
